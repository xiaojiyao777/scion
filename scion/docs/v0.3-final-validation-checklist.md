### Observed Framework Fix Candidates

These issues were observed during the live two-model validation and have been
handled in the validation hardening patch unless explicitly marked P1:

1. [x] `status.json` needs stage-level heartbeat, not only step-level snapshots.
   Long validation/frozen steps must expose current stage, branch, case/seed,
   completed pair count, step start time, and last progress timestamp.
2. [x] Branch lifecycle persistence needs synchronization. `branches.state` must
   reflect abandon/promote/frozen transitions consistently with hypotheses and
   campaign logs.
3. [x] Metrics lineage needs durable DB links. `experiment_events.raw_metrics_ref`
   should point to the corresponding `metrics/*.json` evidence file for every
   experiment event.
4. [x] Expanded screening needs de-duplication and hard accounting. The same
   candidate should not repeatedly rerun identical expanded screening splits
   without a clear budget reason.
5. [x] Long campaigns need partial summaries/checkpoints. A running or interrupted
   campaign should still expose enough structured evidence to audit in-progress
   validation/frozen work and support controlled resume/retry.
6. [ ] P1: Stagnation signals need stronger downstream effect. Plateau/oscillation
   should bias or constrain proposal/scheduler behavior more clearly, instead
   of remaining mostly advisory log output.
