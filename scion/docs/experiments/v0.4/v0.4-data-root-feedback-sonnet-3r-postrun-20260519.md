## 8. Framework 问题与非问题

### Framework 问题

1. Hypothesis premise check 不足。

   agent 已读代码但仍会提出被现有实现部分覆盖的假设。需要在 hypothesis 阶段要求引用具体 baseline evidence，并显式回答“该机制是否已经存在、是否只是参数/顺序变化、当前 telemetry 是否支持该方向”。

2. Declared mechanism telemetry guard 需要更早、更一致。

   Round 1 smoke 通过但 formal 才发现 `alns_stagnation_restart` 不激活。Round 3 smoke 已能挡住 `vns_local_search` activation 缺失。两者说明 smoke/formal 的机制覆盖与 workload probing 仍不一致。

3. Feedback rendering 主次不够清楚。

   `SCREENING_FAIL_WIN_RATE` 应作为第 1、2 轮的 primary decision reason。telemetry guard/warning 应作为辅助诊断，不能在后续 hypothesis context 中与主因混排。

4. Transient code-generation failure 消耗 round budget。

   第 3 轮 fix trace 是外部 502。当前系统把它落为 `structured_output_retry_exhausted` 和 `code generation failed`，并结束 3R。对短实验而言，这会把 infra 抖动误转化为实验轮次消耗。

5. Code prompt budget/manifest 表达仍需整理。

   第 3 轮 retry manifest 里 preview feedback 被标为 omitted，但 raw trace 仍包含 previous failure 摘要。需要让 manifest 精确反映实际进入模型的失败信息，便于审计。

### 非问题

1. `n_steps=3`、`n_experiments=2` 不是统计错误。

   第 3 步进入了 hypothesis/code，但没有生成最终 patch，因此不进入 formal experiment 计数。

2. 本轮 abandon 不是 data-root 假失败。

   两个 formal screening 都是真实 CVRPLIB evaluation，16/16 valid pairs。

3. 第 3 轮不是 Contract/Protocol 架构崩溃。

   Contract preview 已通过，algorithm smoke 正确发现 telemetry activation 缺失。最终失败是 repair LLM 502 后没有恢复。

4. CVRP 内容没有污染 Scion core。

   所有 CVRP-specific algorithm edits 都停留在 branch solver package。

## 9. 下一轮 P0/P1 修复建议

### P0

1. 保留并回归测试 data-root 自动激活。

   当前修复有效。建议把 `SCION_PROBLEM_DATA_ROOT` activation 写入 status/evidence summary，并加一个 CLI regression，确保 formal runner 在未手动传 env 时仍能解析 `cvrplib/...`。

2. 对 code/fix transient 502 增加自动 retry 和分类。

   502、503、504、connection reset 这类外部失败应按 infra transient retry，不应立即消耗实验 round。若最终仍失败，应在 status 中与 model structured-output invalid 区分。

3. 强化 hypothesis premise gate。

   在 hypothesis accept 前要求:

   - 引用具体已读函数/机制证明 gap 存在。

   如果 proposal 声明两个 mechanisms，smoke 至少应在 probe workload 上观察到每个 mechanism 的 activation evidence，或要求 agent 把 conditional/infrequent fallback 降级为 secondary detail，不作为 declared mechanism。

5. Feedback rendering 增加 primary/auxiliary 分层。

   后续 prompt 应突出:

   - primary decision: `SCREENING_FAIL_WIN_RATE`
   - objective summary: win/tie/loss、median delta、runtime ratio
   - auxiliary warnings: telemetry activation/effect

## 10. 是否可以进入 6 轮实验

不建议现在直接进入 6 轮实验。

可以确认的是:

- data-root 修复有效。
- formal screening 已恢复为真实 CVRPLIB evaluation。
- 上一轮 FileNotFound feedback 污染未复现。
- v3 边界控制总体有效，LLM 没有越界修改 Scion core。

但 6 轮实验的目标应是验证 agentic research loop 的算法改进能力，而本轮暴露出的问题会降低 6 轮结论质量:

- agent 仍会误读或弱读 baseline。
- declared mechanisms 与 telemetry/effect 的闭环还不稳定。
- code-generation transient failure 会吞掉轮次。
- feedback 中 primary decision 和 auxiliary telemetry warning 的表达仍可能误导下一轮 hypothesis。

建议路径:

1. 先完成 P0 修复，尤其是 transient retry、hypothesis premise gate、declared mechanism activation contract、feedback 主次分层。
2. 再跑一个 3 轮 sanity experiment，确认没有 FileNotFound、没有 502 吞轮次、每轮 hypothesis 能正确引用 baseline evidence。
3. 若 sanity experiment 至少满足“真实 formal evaluation、无 feedback 污染、无声明机制 activation 缺失、code failure 可恢复”，再进入 6 轮实验。
