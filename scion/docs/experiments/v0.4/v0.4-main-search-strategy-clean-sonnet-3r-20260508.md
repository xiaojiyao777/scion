## Interpretation

This run validates the two intended control repairs:

- `--force-surface main_search_strategy` stayed locked across all rounds.
- C10 singleton semantic novelty no longer blocks distinct
  `main_search_strategy` hypotheses by target file.
- APS compact `read_surface(main_search_strategy)` is usable for the selected
  surface.

It does not validate solver-quality improvement. All candidates failed
screening with `SCREENING_FAIL_WIN_RATE`; no validation, frozen holdout, or
promotion occurred.

The run also did not exercise `route_pair_swap` or `bounded_destroy_repair`.
Therefore the full intended main-search component space remains unvalidated in
formal screening.
