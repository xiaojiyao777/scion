# v0.4 Runtime Smoke, Audit Attribution, And C9c Repair

Date: 2026-05-15

## Scope

This slice repairs the next `solver_design` framework blockers after the first
runtime tie-speedup promotion:

- run `proposal.algorithm_smoke` through a temporary tainted workspace for
  approved `solver_design` algorithm-file patches, not only through in-process
  synthetic preview;
- report the actual solver-design policy file in `solver_algorithm_events`;
- make `context.make_solution(existing_solution)` idempotent in both runtime
  and preview contexts;
- split accepted solver moves into improving and neutral accepted telemetry;
- allow full-algorithm `while within_budget(...)` patterns when the local
  helper's return expression directly references runtime guards.

The original `vrp/` implementation was not modified. All changes are in Scion
framework code, CVRP adapter/spec guidance, tests, and docs.

## Interpretation

This is a framework-control repair, not solver-quality evidence. The positive
signal is that the agent is now producing full algorithm-body changes under
`solver_design`, and Scion can run those changes through a pre-verification
canary without changing promotion rules.

The next bottleneck is making Contract-preview feedback and code repair more
actionable for large algorithm bodies. C9c should continue rejecting true
unbounded loops, but its feedback should identify helper-guarded runtime loops,
collection-progress loops, and counter-bound loops distinctly so the code agent
can repair one category at a time.
