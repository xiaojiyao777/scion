  mostly descriptive rather than operational controls.

That made the earlier direction too close to field exposure and audit
plumbing. Scion still was not controlling the full CVRP solver body.

## Repair

The repair keeps `solver_design` as the top-level CVRP problem object, but
makes `algorithm_body` executable:

- `algorithm_body.baseline_budget_policy` is now explicit. `declared` makes
  `baseline.time_fraction` the actual formal-run budget; `formal_floor`
  preserves the old 0.75 formal guard only when deliberately requested.
- `algorithm_body.phase_sequence` now schedules package-owned components by
  phase. Components whose phase is absent are skipped with
  `algorithm_body_phase_not_enabled`.
- The construction ensemble now records a bounded construction pool, audits
  its distances, and feeds valid non-duplicate construction solutions into
  route-pool recombination.
- `local_cleanup_after_recombination` now runs selected cleanup components
  immediately after an accepted route-pool recombination.
- `adaptive_component_budget` now changes effective component `top_k` by
  instance size and component type, with the effective values recorded in
  runtime evidence.
- Runtime audit now records the repaired controls:
  `main_search_baseline_budget_policy`,
  `main_search_phase_component_order`,
  `main_search_component_top_k_effective`,
  `main_search_construction_pool_size`,
  `main_search_construction_pool_distances`,
  `main_search_local_cleanup_after_recombination`, and
  `main_search_adaptive_component_budget`.

The optional `main_search_phase_unassigned_components` audit remains available
for diagnosis, but it is not a required runtime field because an empty list is
a valid healthy state and selected-surface evidence rejects empty required
values.

together, and APS did not preserve enough deterministic Contract-preview
detail to repair invalid candidates after round 1.

## Runtime-Budget And Preview Repair

The May 13 repair keeps the current `solver_design` direction and makes the
runtime objective explicit at the algorithm-body level:

- `main_search_phase_runtime_ms` and `main_search_elapsed_ms` are valid
  `solver_design` evidence targets in the adapter, problem spec, and solver.
- Prompt/problem-spec guidance states that baseline fraction,
  `route_pool_max_rounds`, component rounds/top-k, route-pool activation
  scope, and adaptive component budgets are runtime/quality controls. Candidate
  timeout or high runtime ratio is a failed algorithm-body design unless the
  quality movement justifies the cost.
- Route-pool sampling and recombination now keep a bounded exit reserve before
  the process timeout. The reserve is large enough for 60s formal runs but
  shrinks on tiny unit-test budgets so local 2s tests do not short-circuit all
  main-search components.
- APS Contract preview now attaches an `issue_summary` to failed preview
  payloads and preserves errors/issues/failed-check details in compact
  observations and self-check failure details. This should turn the previous
  generic Contract-preview loop into actionable repair feedback.
