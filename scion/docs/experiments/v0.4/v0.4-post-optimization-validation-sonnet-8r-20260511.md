  unreadable or skipped. The reserve preserved tool calls, but not enough
  observation-char headroom for the preview payloads.

## Interpretation

This is a useful validation run for the control-plane optimization, but not
solver-quality evidence.

The positive result is that deep-surface discovery improved and ALNS/VNS
attribution is now explainable. R6 proves the new before/after audit fields can
show nonzero selected-surface movement where the prior run reported zero
`alns_vns_phase_delta_sum`.

The negative result is still decisive: none of that movement translated into a
screening pass, and APS preview visibility remains inadequate. The next repair
should focus on compacting preview payloads and stale planner guidance before
another free-surface run. For solver search, the next diagnostics should force
the two still-unexercised mechanism surfaces one at a time:
`destroy_repair_policy` and `route_pair_candidate_policy`.

Do not run long CVRP validation yet.
