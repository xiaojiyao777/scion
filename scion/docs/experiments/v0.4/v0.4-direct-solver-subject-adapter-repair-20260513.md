# v0.4 Direct Solver Subject Adapter Repair - 2026-05-13

## Trigger

The latest 8-round direct solver experiment stayed on `solver_design` and
activated `policies/solver_algorithm.py`, but the candidate behavior was still
too shallow. Screened candidates were mostly `context.baseline(...)` wrappers
with a small post-baseline route edit pass.

Observed screened results:

- 64/64 candidate pairs had `solver_algorithm_active=true`,
  `solver_algorithm_stop_reason=completed`, and no candidate runtime errors.
- No screened candidate passed quality gates.
- Best signal was one case-level win (`win_rate=0.125`) from an Or-opt style
  route edit; all other screened candidates were tie-dominated or mixed.
- Runtime improvements mostly came from shortening baseline budget, not from
  stronger algorithmic search.

Interpretation: the boundary was active, but the adapter still encouraged
baseline-wrapper research instead of exposing a concrete algorithm body for
Scion to study and rewrite.

## Repair

This repair keeps the original `vrp/src` algorithm files read-only and changes
only the Scion CVRP package/adaptation layer.

Implemented changes:

- `policies/solver_algorithm.py` now contains an inactive-by-default
  ALNS/VNS-style full-algorithm template: construction seed pool, capped route
  edit neighborhoods, destroy/repair, perturbation, strict acceptance, runtime
  polling, and solver-algorithm telemetry.
- `_SolverAlgorithmContext` and preview context now expose
  `record_iteration`, `record_move`, and `set_stop_reason` in addition to
  `record_phase`.
- `solver_algorithm_*` runtime evidence now includes search iterations, move
  attempts, accepted moves, best delta, and phase delta/improvement maps.
- Adapter preview rejects shallow `context.baseline(...)` wrappers that do not
  run their own bounded search body and telemetry.
- CVRP solver mechanics and surface prompts now describe
  `policies/solver_algorithm.py::solve(...)` as the direct full-algorithm
  subject. The legacy `main_search_strategy` table is documented as a
  compatibility/regression surface, not the preferred research object.
- Proposal guidance now avoids recommending component-surface switches while
  an active solver-design problem boundary is in force.
- Code-generation retry detail now preserves concrete LLM/APS failure details
  instead of reducing them to `LLM code generation failed`.
- Semantic novelty guidance now exposes the 120-character limit for scalar
  novelty-signature strings.
