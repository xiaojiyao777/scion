## Interpretation

This run is not solver-quality evidence. It only says that the first
boundary-control repair was incomplete:

- Pre-screening solver-design failures were no longer fatal to the boundary.
- Screening solver-design failures still allowed component-policy fallback.
- The active problem object was advisory context, not an enforced proposal
  boundary.
- APS self-check failure was visible in artifacts but not fail-closed.

The follow-up repair therefore makes `solver_design` an active problem-object
constraint whenever the problem declares it and no forced diagnostic surface is
active. Component policies can still be mentioned as implementation hooks or
runtime attribution evidence inside the solver design, but not selected as the
top-level `change_locus`.
