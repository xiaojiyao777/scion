## Summary

The repair is a limited success but not a solver-quality success.

What improved:

- APS hypothesis sessions now saw the intended attribution chain. R2-R5
  feedback included fields such as `main_search_objective_trace`,
  `main_search_objective_delta_by_phase`,
  `main_search_component_accepted_delta_sum`,
  `main_search_component_phase_delta_sum`, and
  `main_search_component_phase_improvement_counts`.
- R4 explicitly used the repaired diagnosis: accepted component deltas were
  nonzero while phase-delta fields were zero.
- Bounded destroy/repair was no longer suppressed by route-pair moves that
  only improved the current perturbed solution. The
  `route_pair_phase_improved` skip reason disappeared in this run.
- R1-R4 had complete selected-surface runtime audit: 64/64 candidate-side pairs
  carried all 51 required `main_search_strategy` fields.

What did not improve:

- R1-R4 all still failed `SCREENING_FAIL_WIN_RATE`.
- Component-local accepted moves still did not refresh phase best. In R1, R3,
  and R4, accepted deltas were positive, but phase-delta sums,
  phase-improvement counts, and `improvement_loop` phase delta remained zero.
- Screening quality regressed versus the prior smoke. R1-R4 mean case win rate
  fell to `0.03125`, with max `0.125`.
- R5 drifted off the forced surface and proposed `create_new/route_local` for
  `operators/oropt_intra_route.py`. The forced-surface guard correctly stopped
  it before code generation, but the diagnostic did not produce a fifth
  on-surface candidate.

No validation, frozen, promotion, or final-evidence signal was produced.
`formal_ready=false`.

## Interpretation

The 17f89de slice should be accepted as a diagnostic/control repair:

- selected-surface runtime attribution is visible to APS;
- BDR suppression by non-phase-best route-pair accepts is fixed;
- selected-surface runtime audit is complete for all screened candidates;
- Decision remains insulated from tainted attribution payloads;
- budget and C10 did not regress for R1-R4.

It is not an efficacy repair:

- all screened candidates failed `SCREENING_FAIL_WIN_RATE`;
- phase-level component improvement stayed zero;
- case-level quality regressed versus the prior smoke;
- R5 showed forced-surface prompt/control hygiene is still imperfect.
