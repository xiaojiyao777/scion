inert helpers.

## Repair

The repair keeps the solver-design boundary unchanged:

- objective semantics, constraints, parser behavior, protocol splits, seeds,
  time-limit enforcement, and Decision thresholds remain outside candidate
  control;
- candidates still modify branch-owned solver-design modules or the stable
  entrypoint, not the original `vrp/` algorithms;
- proposal observations and smoke evidence remain tainted and
  non-promotional.

Changes:

- Hypothesis context now expands solver-design wildcard targets such as
  `policies/baseline_modules/*.py` into concrete module paths from the
  champion snapshot. This lets the model choose
  `construction.py`, `destroy_repair.py`, `local_search.py`, `acceptance.py`,
  or `scheduler.py` explicitly instead of defaulting to the wildcard or the
  scheduler.
- After repeated recent win-rate-zero scheduler screening failures,
  hypothesis guidance now asks for target diversity: choose the file that owns
  the mechanism, and use scheduler only as orchestration or wiring.
- Solver-design code prompts now state that scheduler edits should not be
  another phase-order/weight/runtime knob unless they change the actual
  bounded search trajectory.
- Local-search prompts now tell the code agent to integrate new move operators
  through the existing `_default_vns_operators()` / `_vns(...)` path and not to
  invent detached scheduler `_run`/`run` entrypoints.
- C9e now treats loaded function-name references as integration edges, not
  only direct `Call` nodes. This recognizes first-class operator lists such as
  `_default_vns_operators()` while still rejecting helpers that are never
  reachable from the branch solver path.

## Interpretation

This is framework-positive and solver-quality-negative.

The framework improvement is real: APS can now select concrete branch-owned
solver modules, completed code sessions can modify those modules, and C9e no
longer blocks a legitimate VNS operator-list integration pattern. Scion still
keeps the boundary and audit path intact: failed candidates reached official
screening only after Contract, algorithm smoke, and Verification, then were
abandoned by protocol evidence.

The solver-quality result is still weak. The latest local-search candidate
made the algorithm slower and worse on formal screening. That should feed the
next research attempt as algorithm evidence, not be treated as a framework
failure or as a reason to return to singleton policy exposure.
