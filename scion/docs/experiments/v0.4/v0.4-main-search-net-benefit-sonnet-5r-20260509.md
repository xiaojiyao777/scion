## Comparison To Bounded Destroy/Repair Smoke

The previous five-round smoke from commit `8a37fd6` held
`force_surface=main_search_strategy` for all rounds, screened four candidates,
and reached a best case-level win rate of `0.25`. Its strongest candidate had
`bounded_destroy_repair` accepted 21 moves, positive best deltas in 12/16
pairs, and 157 repair fallbacks.

This run from commit `8d49abe` regressed on screening quality:

- one forced diagnostic round drifted to `route_local`;
- screened candidates dropped to three;
- all official case-level win rates were `0.0`;
- bounded destroy/repair accepted zero moves;
- route-pair accepted moves did not become case-level wins.

The net-benefit guard reduced uncontrolled destroy/repair acceptance and
changed skip behavior. It did not produce net case-level efficacy.

## Interpretation

This run is valid control-path and diagnostic evidence, not solver-quality
evidence. It does not satisfy the condition for long validation.

The current blocker is now twofold:

- forced-surface diagnostic control regressed, allowing a `route_local`
  operator candidate in a `main_search_strategy` smoke;
- the net-benefit guard did not translate accepted route-pair movement or
  guarded destroy/repair behavior into case-level screening wins.

The `main_search_baseline_param_clamps` runtime field also needs a stable
representation when no clamp is present, because an empty required field
correctly fails selected-surface runtime audit but makes some candidates
unusable for screening.

## Next Step

Do not run a long CVRP validation yet.

Recommended next slice:

- make `--force-surface main_search_strategy` a hard proposal constraint for
  forced diagnostics so operator surfaces cannot drift into the run;
- make `main_search_baseline_param_clamps` consistently non-empty or adjust the
  declared runtime evidence contract to represent no-clamp status explicitly;
- diagnose why accepted `route_pair_swap` moves do not become case-level wins;
- redesign bounded destroy/repair around final returned objective benefit, not
  only component-local accepted moves;
- keep using five-round forced diagnostics until at least one candidate moves
  beyond the previous `0.25` case-level win-rate ceiling or approaches the
  screening threshold.
