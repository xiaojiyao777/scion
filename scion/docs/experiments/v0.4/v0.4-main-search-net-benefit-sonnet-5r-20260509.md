deeply using screening/runtime feedback as a research signal; it was mostly
searching over surface guidance and bounded strategy knobs.

## Interpretation

This run is valid control-path and diagnostic evidence, not solver-quality
evidence. It does not satisfy the condition for long validation.

The current blocker is now twofold:

- forced-surface diagnostic control regressed, allowing a `route_local`
  operator candidate in a `main_search_strategy` smoke;
- the net-benefit guard did not translate accepted route-pair movement or
  guarded destroy/repair behavior into case-level screening wins.

The deeper APS analysis makes the first blocker sharper: the drift happened in
the hypothesis phase. The command and launch artifacts carried
`force_surface=main_search_strategy`, but the hypothesis task and tool context
still allowed the agent to choose from all declared surfaces. Code generation
then faithfully implemented the already-approved off-surface `route_local`
hypothesis.

The run also shows that APS is not yet doing deep research over the historical
evidence. The agent mostly read problem and surface context, did not call
memory or screening/runtime feedback tools in the analyzed artifacts, and did
not appear to use the previous `8a37fd6` round-5 lesson directly. Its
`main_search_strategy` proposals were bounded component/parameter combinations
around baseline fraction, enabled components, rounds, `top_k`, restart, and
perturbation. That is useful surface search, but it is not yet deep algorithm
diagnosis.

The `main_search_baseline_param_clamps` runtime field also needs a stable
representation when no clamp is present, because an empty required field
correctly fails selected-surface runtime audit but makes some candidates
unusable for screening.

## Next Step

Do not run a long CVRP validation yet.

Recommended next slice:

1. Fix APS/core forced-surface propagation first. A forced diagnostic should
   make the hypothesis task, tool guidance, draft validation, and finalization
   reject any surface other than the requested surface.
2. Fix `main_search_baseline_param_clamps` evidence semantics. Either emit an
   explicit non-empty no-clamp status or remove the field from required
   evidence when an empty object is a valid no-op value.
3. Make APS actually use bounded screening/runtime feedback in hypothesis
   sessions for forced diagnostics. The next agent should see why prior rounds
   failed, not infer from a truncated gate label.
4. Diagnose and repair the CVRP strategy semantics behind round 3: accepted
   `route_pair_swap` moves must correspond to final returned objective benefit
   or be labeled differently from net accepted improvements.
5. Redesign bounded destroy/repair around final returned objective benefit, not
   only component-local accepted moves.
6. Keep using five-round forced diagnostics until at least one candidate moves
   beyond the previous `0.25` case-level win-rate ceiling or approaches the
   screening threshold.
