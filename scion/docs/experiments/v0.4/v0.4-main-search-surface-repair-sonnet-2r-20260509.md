## Interpretation

This run validates:

- clean committed execution from `e25680a`;
- persistent forced `main_search_strategy` selection;
- candidate-side selected-surface runtime audit for all declared fields;
- the new component telemetry path for selected / attempted / accepted /
  skipped main-search components;
- nontrivial pair-level screening movement and faster median runtime for the
  one evaluated candidate.

It does not validate the intended deep-component research slice:

- `route_pair_swap` was not selected or attempted;
- `bounded_destroy_repair` was not selected or attempted;
- only one candidate reached screening;
- round 2 was blocked by C10 duplicate detection;
- no validation, frozen holdout, promotion, or final evidence refs were
  produced.

Do not start a long CVRP solver-quality validation from this result.

  even after the structured novelty repair;
- rerun a short forced smoke only after the above, and move to longer
  validation only if deep components are selected, attempted, audited, and still
