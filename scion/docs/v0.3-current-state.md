### 5.3 F2 暴露的更深问题：v3 lex-order 与 ci_low 度量错配

**sonnet s11 仍 0-promote 的根因不是 T1+T2+T3 修不对**，是暴露出更深的设计层问题。

具体表现（branch c877f314 frozen 阶段）：
- 12/12 frozen pairs 全部 cmp=win 且 decisive=subcategory_splits（splits 主轴 clean improvement）
- median cost delta = +4605（次轴中位数也是正向）
- 但因 3 个大负 outlier（-19890, -7492, -6295）让 cost delta 的 bootstrap CI 跨 0
- → **`ci_low<0` → FROZEN_FAIL**

设计框架审查：
- v3 §8.6 frozen gate "ci_low >= 0" 未规定 metric
- `scion/protocol/stats.py` 实现 `bootstrap_ci(deltas)` 中 `deltas` 是 **total_cost 差**，即 lex-order **次轴**
- v3 设计意图（splits 主、cost 次） vs 实现实际（gating on cost CI）= 系统性偏离
- 该候选在 v3 lex-order 下应 promote，被实现误杀

附加问题：
- sonnet s11 R25 出现一个干净 SCREENING_PASS（wr=0.7, md=+451）但**与早停同秒触发**，validation 一次都没跑就被 kill。早停只看 idle 不感知 queue 中候选。
- 详见 `docs/v0.3-f2-analysis.md §四 / §五`

后验分析固定检查：

3. 每个 promoted candidate 的 evidence：validation/frozen win rate、median delta、decisive objective 分布、case/seed 稳健性。
4. 失败画像：heavy/light verification、soft abandon、abandon_fast、stagnation signal、重复 screening。
5. 框架行为是否符合 v3/v0.3：branch 内迭代、stage gate、frozen holdout、generic objective comparison、adapter boundary、failure routing、lineage auditability。
6. 是否满足启动 6-campaign rerun 的实验和工程 gate。

### 8.3 已处理的 P0 优化项

这些项来自两模型 validation 的完整后验分析，已在当前代码中落地：

1. **`status.json` stage-level heartbeat** ✅
   长 validation/frozen step 需要写当前 stage、branch、case/seed、已完成 pair 数、step start、last progress timestamp。

2. **branch lifecycle persistence sync** ✅
   `branches.state` 需要和日志/hypothesis 状态同步，避免已 abandon/promote 的 branch 在 DB 中仍显示 `explore` 或 `ready_frozen`。

3. **metrics lineage DB link** ✅
   `experiment_events.raw_metrics_ref` 需要指向对应 `metrics/*.json`，否则后验审计只能靠日志和文件名重建。

4. **expanded screening de-dup/accounting** ✅
   同一候选不应在相同 expanded screening split 上重复跑出完全相同统计；需要去重、硬上限和明确 budget reason。

5. **partial summary / checkpoint / controlled resume** ✅
   长 campaign 运行中或中断时也应有结构化 partial evidence，至少能审计 in-progress validation/frozen，并支持受控恢复。

6. **weight revision invalidation audit** ✅
   weight-only update 落地时记录 active branches、stale branch ids 和 revision 信息；除 in-flight frozen 外的 active branches 会 conservative reconcile。

7. **stagnation signal downstream effect**
   plateau/oscillation 目前仍主要通过 forced locus diversification 起作用。更强的 family/action scheduler bias 保留为 P1，不阻塞 v0.3 完整实验。
