## 四、T1+T2+T3：按 v3 第一性原理重校准决策规则（commit `4e797b1`）

首轮失败后不是打补丁修 A1 的实现，而是回到 v3 蓝图 §8.6 / §8.7 / §11.5 重新推导规则。三个子修正作为一个设计整体：

### 4.1 T1 — 撤销 A1 SPND cap

`core/decision.py`：`wr >= threshold AND md < 0` → `QUEUE_VALIDATE`（无 cap）。

依据：v3 §8.6 lex-order，screening 确定性 cases 的 md 信号不足以替代 validation bootstrap CI；F 数据实测 SPND 候选在 frozen 经常 md>0 → promote（s29/s47）。

### 4.2 T2 — VALIDATION_EXPAND_EXHAUSTED md guard

`core/decision.py` validation 分支：
- `validation_expand_count >= 1 AND ci_low<0 AND md<0` → **ABANDON**（`VALIDATION_EXPAND_EXHAUSTED_FAIL`）
- `validation_expand_count >= 1 AND ci_low<0 AND md>=0` → QUEUE_FROZEN（`VALIDATION_EXPAND_EXHAUSTED_MARGINAL_PASS`）

把"防浪费 frozen 预算"职能**从 screening 层迁到 validation 层**（信号更可靠）。闭合原 §七.1 open question（validation 不再无条件升 frozen）。尊重 v3 §8.6 validation gate 三条件。

### 4.3 T3 — stage-aware per-candidate expand counters

- `Branch` 字段：`expand_count` → `screening_expand_count` + `validation_expand_count`
- `campaign.py` 增量：state == EXPLORE_EXPAND → screening_expand_count += 1；state == VALIDATING_EXPAND → validation_expand_count += 1
- `_run_explore_step` 在 `pending=None`（新 hypothesis）时重置两个计数器为 0
- `DecisionFeatures` 同步新增两字段；决策规则三处从 `expand_count` 改为对应 stage 字段

依据：v3 §11.5 "screening expand: 1 次 / validation expand: 1 次" 是 **per-candidate** 预算，不是 per-branch 共享。原单计数器是实现 bug。

### 4.4 测试新增

`tests/test_decision.py`：+10 新测试（T1 unaffected-by-counter x2、T2 FAIL/MARGINAL_PASS、T3 unblocked/extract-propagate）；2 个 BORDERLINE 测试改字段；删除原 SPND_EXHAUSTED 测试。
`tests/test_campaign.py`：+1 integration `test_new_hypothesis_resets_expand_counters`。
