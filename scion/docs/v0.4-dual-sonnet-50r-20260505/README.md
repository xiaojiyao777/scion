## Executive Interpretation

The framework behaved much better than the earlier 2026-05-04 dual run:

- fixed-round validation was honored;
- runtime evidence was recorded;
- candidate runtime failure failed closed;
- CVRP research surfaces were exposed to the LLM;
- warehouse promotion, frozen budget governance, and async weight optimization
  were exercised.

The run also exposes remaining v0.4 gaps:

- CVRP still needs broader algorithm research surfaces beyond simple
  post-baseline operators plus a first `search_policy` surface.
- C10 novelty is too coarse for singleton policy surfaces: four distinct
  `search_policy` attempts were blocked as duplicate because the key collapses
  to `(surface, action, target_file)`.
- CVRP final state left one active branch after max-round stop, which is a
  lifecycle cleanup issue.
- Both summaries are still `formal_ready=false` because `final_evidence_refs`
  are missing.
- Warehouse promotion paths are reconstructable, but promoted champion rows
  still have `promotion_experiment_id=NULL`.
- Both campaigns contain unreferenced `v8_run*` metrics noise that should not
  pollute future evidence manifests.

## Validity Judgment

This run is valid as a v0.4 framework and campaign-dynamics validation. It is
not valid as final solver-quality evidence because final quality/runtime
evidence refs were not produced.

CVRP's 0 promotion should be interpreted as:

```text
surface/model/search-space limitation + screening gate discipline
```

not as:

```text
experiment crashed
```

Warehouse's 3 promotions should be interpreted as:

```text
the branch-governed v0.3 problem path still works after v0.4 framework changes
```

not as:

```text
all v0.4 formal evidence requirements are closed
```
