## P4-01 CVRPLIB Input Support

Added:

```text
scion/scion/problems/cvrp/cvrplib.py
scion/scion/tests/test_cvrp_cvrplib_adapter.py
```

Updated:

```text
scion/scion/problems/cvrp/adapter.py
```

### Semantics

- `CvrpAdapter.load_instance()` now dispatches:
  - `.json` to existing `CvrpInstance.from_json`;
  - `.vrp` to Scion's owned CVRPLIB parser.
- Parser supports EUC_2D only.
- Parser reads:
  - `NAME`;
  - `DIMENSION`;
  - `CAPACITY`;
  - `EDGE_WEIGHT_TYPE`;
  - `NODE_COORD_SECTION`;
  - `DEMAND_SECTION`;
  - `DEPOT_SECTION`.
- CVRPLIB 1-based ids are mapped into Scion zero-based ids with depot `0`.
- Sibling `.sol` files populate `bks` and `bks_routes` when present.
- Missing `.sol` leaves reference fields unknown.
- Unsupported `EDGE_WEIGHT_TYPE` fails closed.
- Integer-distance mode is used for integer coordinates; decimal coordinates
  keep continuous Euclidean distance.

Tests use temporary synthetic `.vrp/.sol` files only. They do not read raw
`vrp/cvrplib/**` files.

## P4-04b CVRP Operator Runtime Boundary

Added:

```text
scion/scion/problems/cvrp/registry.yaml
scion/scion/tests/test_cvrp_solver_operator_runtime.py
```

Updated:

```text
scion/scion/problems/cvrp/solver.py
```

### Semantics

- CVRP solver now reads `--registry` when the file exists.
- Missing or empty registry preserves the original nearest-neighbor baseline.
- Operators are loaded from workspace-relative `file_path` plus `class_name`.
- Operator paths that escape the workspace are skipped and audited.
- Operator loading/runtime errors are counted in solver `runtime` metadata.
- Operators run in a deterministic bounded loop:
  - sorted by descending weight and registry order;
  - capped by a fixed round count and the solver time budget.
- Operators must return `CvrpSolution`.
- Returned solutions must pass adapter consistency and feasibility checks.
- Acceptance uses only the promotion objective:
  - first `fleet_violation`;
  - then `total_distance`.
- BKS/gap is not used in runtime acceptance.
- Solver output keeps the existing JSON contract:
  - `routes`;
  - `feasible`;
  - `objective`;
  - `runtime`.
- Runtime metadata now includes operator audit counters/events.
- Added root `registry.yaml` with an empty operator list so branch workspaces can
  register generated `create_new` operators.

This closes the main runtime gap found after P4-04a: generated CVRP operators
can now affect evaluated solver output.

## 2026-05-04 Post-Audit Formal-Readiness Fix

### Trigger

A real CVRP Sonnet run completed with all screening pairs tied and no promotion.
Detailed audit showed it was not valid quality evidence:

- hypothesis/code prompts still described the old high-invocation VNS pool;
- the CVRP wrapper was using a small post-baseline operator loop;
- generated operators commonly returned workspace-local `models.CvrpSolution`,
  which failed package-class `isinstance` checks;
- the formal run did not use the intended repo `vrp/src` ALNS+VNS baseline.

### Framework Boundary Repair

- Removed research-object mechanics from framework prompt text.
- Added generic `solver_mechanics` plumbing from adapter to proposal prompts.
- Kept CVRP-specific mechanics only in `scion.problems.cvrp.adapter`.
- Removed CVRP-specific env-var naming from framework subprocess env handling;
  the runner now passes generic `SCION_*` variables.
- Removed warehouse-object reconstruction from framework feasibility/objective
  verification fallback. New problems must use `ProblemAdapter`; non-adapter
  legacy problems need generic oracle hooks.

### CVRP Runtime Repair

- CVRP `.vrp` runs use repo `vrp/src` ALNS+VNS baseline when
  `SCION_PROBLEM_DATA_ROOT` points at the repo `vrp` directory.
- Synthetic `.vrp` fixtures do not implicitly trigger the full baseline.
- Required-baseline fallback is a runtime audit failure.
- Workspace-local `CvrpSolution` return values are accepted structurally when
  route data is valid.
- Malformed, infeasible, or exception-raising operators fail closed through
  runtime audit instead of becoming ties.
- `vrp/src` internal node ids are mapped back to Scion's depot-first CVRP ids.

### Governance Repair

- Runtime audit failures now include operator invalid outputs and required
  baseline failures.
- Protocol raw metrics include candidate/champion runtime summaries.
- Validation/frozen exposed summaries remain aggregate-only and no longer expose
  raw metrics paths.
- Missing canary configuration fails closed in the evaluation pipeline.
- `V9_perf_guard` uses the configured runtime ratio threshold.
- `ProblemSpecV1` can be loaded as the authoritative source and bridged to the
  legacy runtime shape.
