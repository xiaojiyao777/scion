## Purpose

Validate that the Scion v0.3 closure optimization (see `scion/docs/v0.3-optimization-design.md`) restores promotion cadence to W16 baseline after the post-opt regression was identified on 2026-04-23.

Not intended as a W17 mechanism study; purpose is to **confirm the regression is fixed** and the campaign architecture remains sound under the refactored CampaignManager (B1/B2/B3 service extractions).

## Hypothesis under test

The combination of:
- A1 (SPND expand_count cap)
- A2 (idle counter skips frozen_fail/validate steps)
- A3 (stagnation_window 15→25)
- A4 (BORDERLINE md>=0 guard — added after smoke revealed parallel leak)
- B1-B3 (service extraction — no behavioral change expected)

will restore Sonnet's promotion cadence on `synthetic` to at least W16 level (avg 2.3 promotions per campaign).

## Success criteria

| Metric | Target | Derived from |
|---|---|---|
| Sonnet synthetic × 3 seeds avg promotions | **≥ 2.0** | W16 achieved 2.3 (s11=2, s29=3, s47=2) |
| GPT-mini synthetic × 3 seeds avg promotions | **≥ 1.5** | W16 achieved 1.7 (s11=2, s29=2, s47=1) |
| Early-stop triggered round (if at all) | **≥ 25** | post-opt Sonnet fired at round 15 with old stagnation_window=15 |
| BORDERLINE_NEGATIVE_DELTA reason present in any branch | **true is fine** | Verifies A4 actually fires in realistic scenarios |
| SPND_EXHAUSTED reason present in any branch | **true is fine** | Verifies A1 cap activates |
| All 6 campaigns complete without crash | **required** | Framework regression check post B1/B2/B3 |

**Failure interpretation**:
- If Sonnet < 2.0 avg promotions: either A1-A4 insufficient, or B1-B3 regression. Investigate by replaying one campaign with A1-A4 absent.
- If any campaign crashes: likely B-side service extraction bug. Preserve logs, open postmortem.
- If BORDERLINE_NEGATIVE_DELTA / SPND_EXHAUSTED never fires: not a failure by itself, but means we can't cheaply verify those code paths.
