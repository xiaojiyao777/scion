# Scion v0.4 Phase 3 Closeout

*Date: 2026-05-01*
*Branch: v0.4-dev*

## Summary

Phase 3 completed the final quality evidence infrastructure required before
formal CVRP campaigns.

The phase deliberately kept final quality evaluation outside the campaign loop.
Campaign promotion evidence remains separate from final quality/runtime
evidence.

## Completed Components

### Generic Final Quality Writer

Added:

```text
scion/scion/evidence/final_quality.py
```

Responsibilities:

- build final quality/runtime/failure summaries from in-memory case records;
- write the six-file evidence package;
- count timeout, crash, error, infeasible, runtime regression, and benchmark
  incomparability;
- keep CVRP BKS/gap/route-count fields report-only.

### CVRP CSV Import Path

Added:

```text
scion/scion/evidence/cvrp_baseline_import.py
scion/scion/evidence/cvrp_package.py
```

Responsibilities:

- translate existing `vrp/results/full_experiment_seed0_final.csv`-style rows
  into `QualityCaseRecord`;
- build/write a no-run final evidence package from baseline/candidate CSV
  artifacts;
- treat CSV `path` fields as opaque artifact identifiers.

### Runner-Backed CVRP Final Evaluation

Added:

```text
scion/scion/evidence/cvrp_final_evaluation.py
```

Responsibilities:

- evaluate baseline and candidate workspaces on identical case/seed/time-limit
  sets;
- use the CVRP adapter for instance loading, solver output deserialization,
  consistency, feasibility, and objective recomputation;
- preserve failed or incomplete runs as non-comparable evidence rows.

### Case Manifest Path

Added:

```text
scion/scion/evidence/cvrp_case_manifest.py
scion/scion/evidence/cvrp_manifest_evaluation.py
```

Responsibilities:

- build deterministic fixed evaluation manifests from CVRP result CSV rows;
- filter by subset, BKS, BKS route count, benchmark comparability, status, and
  feasibility;
- convert manifests into runner-backed final evaluation configs.

### Campaign Summary Ref Attachment

Added:

```text
scion/scion/evidence/final_evidence_refs.py
```

Responsibilities:

- convert final evidence package result refs into a stable
  `final_evidence_refs` payload;
- attach refs through `EvidenceRecorder.attach_final_evidence_refs()`;
- avoid changing per-step summary schema.

## Evidence Package Files

Phase 3 standardizes this package:

```text
evidence_manifest.json
final_quality.json
final_quality.csv
per_case_quality.csv
runtime_summary.json
failure_summary.json
```

## Safety Boundaries

Phase 3 did not:

- read raw `vrp/cvrplib/**` benchmark files in tests;
- run LLMs;
- change promotion decisions;
- make campaign execution automatically run final evaluation;
- move BKS/gap into promotion logic;
- trust solver-reported CVRP distance in runner-backed final evaluation.

## Phase 4 Readiness

Phase 4 can now focus on formal CVRP campaign planning and execution.

The intended path is:

```text
CVRP result CSV
  -> fixed case manifest
  -> manifest-driven final evaluation after campaign
  -> final evidence refs attached to campaign summary
```

Recommended Phase 4 first steps:

1. Define the exact CVRP screening/validation/frozen/final case manifests.
2. Decide the first formal campaign matrix and runtime budgets.
3. Run one short end-to-end CVRP campaign and then run manifest-driven final
   evidence as a post-campaign step.
4. Only after that, run the multi-model/multi-seed formal matrix.
