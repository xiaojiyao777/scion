## Data Root

Formal runs must set:

```bash
export SCION_CVRP_DATA_ROOT="$PWD/vrp"
```

`scion.problems.cvrp.solver` resolves missing relative instance paths through
this environment variable. `LocalSubprocessRunner` now whitelists the variable
for solver subprocesses.

This avoids copying benchmark files into branch/champion workspaces.

## Campaign Command

From `scion/`:

```bash
SCION_MODEL=<model> \
SCION_CVRP_DATA_ROOT="$(cd .. && pwd)/vrp" \
/home/clawd/miniconda3/envs/claw/bin/python -m scion.cli.main run \
  --problem scion/problems/cvrp/problem.yaml \
  --protocol scion/problems/cvrp/formal/protocol.yaml \
  --split scion/problems/cvrp/formal/split_manifest.yaml \
  --seeds scion/problems/cvrp/formal/seed_ledger.yaml \
  --campaign-dir <output-root>/<model>_seed<seed>/campaign \
  --rounds 100 \
  --time-limit-sec 5
```

Use the matrix runner to set `SCION_MODEL` and campaign seed once campaign
seeding is wired at the launcher level. Until then, the seed is a campaign
directory label and the protocol seed ledger is the deterministic evaluation
source.

## Final Evidence

Final evidence remains a post-campaign manual step.

Configuration:

```text
manifest: scion/problems/cvrp/formal/manifests/final.json
baseline_workspace: scion/problems/cvrp
candidate_workspace: <campaign>/champions/<latest>
baseline_registry_path: scion/problems/cvrp/registry.yaml
candidate_registry_path: <campaign>/champions/<latest>/registry.yaml
time_limit_sec: 10
seeds: 0, 1, 2
```

The final evidence package must write:

```text
evidence_manifest.json
final_quality.json
final_quality.csv
per_case_quality.csv
runtime_summary.json
failure_summary.json
```

Then attach refs to `campaign_summary.json` through
`attach_final_evidence_package`. Do not mutate per-step summary schema.

## Promotion Policy

Campaign promotion objective remains:

```text
1. fleet_violation
2. total_distance
```

BKS, gap percentage, and BKS route count are final-report evidence only. They
must not enter promotion or operator runtime acceptance.

## Residual Risks

- The current CLI does not expose a campaign random seed option. Matrix seed
  labels are ready, but launcher-level seed injection still needs a small
  implementation pass if model calls or scheduling need deterministic seed
  separation.
- The formal manifests intentionally exclude XML rows without complete BKS and
  BKS-route evidence; this is correct for first formal reporting, but it limits
  coverage of very large instances.
- Runtime budgets are initial values. If timeout/failure summaries show high
  incompleteness, increase budgets before interpreting quality.
- Final evidence must be run after campaign completion; it is not part of the
  campaign loop.
