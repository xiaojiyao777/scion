scion/docs/experiments/v0.4/v0.4-dual-sonnet-50r-20260505/

## Goal

Repair framework-level defects exposed by the 50-round CVRP and warehouse
campaigns before running another long experiment.

This is not the P1 research-object surface expansion. Do not add new
VRP-specific or warehouse-specific heuristic logic to Scion core. The purpose
is to make Scion's generic research-surface governance, evidence lineage, and
campaign closeout conform more tightly to the v3/v0.4 design.

## Scope

### P0-A: Surface-Aware Semantic Novelty

Problem observed:

```text
C10_novelty key for singleton policy surfaces collapsed to
(search_policy, modify, policies/search_policy.py)
```

This blocked semantically different policy hypotheses in CVRP rounds 11, 20,
41, and 50.

Required behavior:

- Preserve strict duplicate blocking for ordinary operator `modify/remove`
  hypotheses.
- For `research_surfaces.kind == "policy"`, build a semantic novelty key that
  includes a normalized/fingerprinted hypothesis intent in addition to
  `(surface, action, target_file)`.
- Identical policy hypotheses should still fail C10.
- Distinct policy hypotheses against the same singleton file should pass C10.
- The implementation must stay problem-agnostic; do not hardcode
  `search_policy` or CVRP names except in tests/fixtures.

Likely files:

```text
scion/scion/contract/gate.py
scion/scion/tests/test_contract.py
scion/scion/tests/unit/test_research_surfaces.py
```

### P0-B: Decision Reason Codes in Summary

Problem observed:

CVRP round 42 had a candidate timeout. DB decision lineage recorded
`CANDIDATE_RUNTIME_FAILURE`, but `campaign_summary.json` primarily exposed the
protocol gate's `SCREENING_FAIL_WIN_RATE`, making post-run audit harder.

Required behavior:

- Preserve protocol gate reason codes.
- Also expose DecisionEngine reason codes in `campaign_summary.json`.
- For protocol rows, include an unambiguous effective/decision reason field so
  runtime vetoes are visible without reading SQLite.
- Do not let free-text LLM rationale enter Decision or summary reason
  derivation.

Likely files:

```text
scion/scion/core/evidence_recorder.py
scion/scion/tests/unit/core/test_evidence_recorder.py
```

### P0-C: Promotion Lineage on Champion Rows

Problem observed:

Warehouse promoted champion rows had `promotion_experiment_id=NULL`. Promotion
paths were reconstructable from summary/events, but v3 says champion state
should include `promotion_experiment_id`.

Required behavior:

- Structural champion rows created by promotion must persist a non-empty
  `promotion_experiment_id`.
- That id should correspond to the promotion experiment event row when
  practical.
- Weight-only revisions should preserve the original structural champion's
  promotion id.
- Initial v1 champion may keep `promotion_experiment_id=None`.

Likely files:

```text
scion/scion/core/decision_finalizer.py
scion/scion/core/evidence_recorder.py
scion/scion/core/promotion_service.py
scion/scion/core/promotion_lifecycle.py
scion/scion/core/campaign.py
scion/scion/tests/unit/core/test_promotion_service.py
scion/scion/tests/test_campaign.py
```

Implementation note:

The cleanest route is to generate the promotion event id before committing the
champion, attach it to the `ChampionState`, and pass the same id into lineage
recording. Avoid post-hoc DB mutation if possible.

### P0-D: Max-Round Branch Closeout

Problem observed:

CVRP stopped at `max_rounds_exhausted` with one active `explore` branch. This
does not invalidate metrics, but it leaves final lifecycle state ambiguous.

Required behavior:

- When the campaign stops because `max_rounds_exhausted`, close or explicitly
  terminalize active branches.
- Preferred behavior: mark remaining active branches `ABANDONED` with a
  failure/status code such as `MAX_ROUNDS_EXHAUSTED`, persist branch store, and
  leave promoted/abandoned branches untouched.
- Final `status.json` and `campaign_summary.json` should then report
  `n_active_branches=0`.
- Do not close branches when stopping due to circuit breaker, API balance, or
  other non-budget interruptions unless an existing policy says so.

Likely files:

```text
scion/scion/core/campaign_loop.py
scion/scion/core/campaign_composition.py
scion/scion/core/campaign.py
scion/scion/core/branch.py
scion/scion/tests/test_campaign.py
```

## Acceptance Tests

At minimum, add or update focused tests for:

1. Two distinct policy-surface modify hypotheses against the same target file
   pass C10 novelty.
2. An identical policy-surface modify hypothesis still fails C10 novelty.
3. Ordinary operator modify novelty remains strict by `(locus, action,
   target_file)`.
4. Campaign summary exposes DecisionEngine runtime veto reason codes separately
   from protocol gate reason codes.
5. Promoted champion rows have a non-empty `promotion_experiment_id` and the
   lineage event with that id exists.
6. `run(max_rounds=N)` closeout leaves no active branches when the stop reason
   is `max_rounds_exhausted`.
