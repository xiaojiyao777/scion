```text
failure_summary.json
```

## Semantics

The package writer keeps final reporting separate from promotion decisions:

- BKS/gap/route-count fields are report-only.
- CVRP benchmark comparability is separate from CVRP feasibility.
- Missing `bks_routes` produces unknown benchmark comparability, not `true`.
- Timeouts, crashes, errors, infeasible rows, and benchmark-incomparable rows
  are preserved and counted.
- Runtime regression is computed from a configurable ratio threshold.

During main-session review, the comparison boundary was tightened:

- if either baseline or candidate has a failure category, `comparison` is forced
  to `not_comparable`, even if a caller supplied `better`, `tie`, or `worse`;
- unknown runtime error categories such as `oom` count toward `n_error` while
  the original `error_category` field remains visible in the row.

This avoids final evidence accidentally reporting a timeout/crash as a win.

## P3-02 CVRP Baseline CSV Importer

```text

```


Public API:

```text
CvrpCsvResultRow
load_cvrp_result_rows
build_cvrp_quality_records
load_cvrp_quality_records
```

The importer translates `vrp/results/full_experiment_seed0_final.csv`-style
rows into `QualityCaseRecord` rows. It does not write evidence files directly;
P3-01 remains the only writer for the final quality package.

### Import Semantics

- `path` is preserved only as an opaque CSV artifact string and is never opened.
- Rows are aligned by `(case_id, seed)` for baseline-vs-candidate comparison.
- `candidate_rows=None` creates baseline self-comparison records.
- Self-comparison sets `comparison="equal"` only when the row is valid and has
  a cost; failed or incomplete rows are left for the writer to classify as
  `not_comparable`.
- Missing candidate rows become candidate error records.
- Extra candidate rows without a matching baseline fail closed.
- `wall_time` is preferred over `time`; both are interpreted as seconds and
  converted to milliseconds.
- Missing numeric fields become `None`, not `0`.
- Missing `bks_routes` does not imply benchmark comparability.

### Semantics

- Baseline-only CSV produces self-comparison records through the importer.
- Paired baseline/candidate CSVs preserve importer alignment and failure
  semantics.

  effects.

- Caller-provided campaign id, labels, runtime threshold, and objective
  tolerance propagate into manifest, final quality, and runtime summary files.
- Opaque `path` fields inside CSV rows remain unopened.

## P3-03b Runner-Backed CVRP Final Evaluation Service

Added:

```text
scion/scion/evidence/cvrp_final_evaluation.py

```

Updated:

```text
scion/scion/evidence/__init__.py
scion/docs/planning/v0.4/v0.4-phase3-task-manifest.md
```

Public API:

```text
CvrpFinalEvaluationConfig
CvrpSideResult
evaluate_cvrp_final_quality_records
build_cvrp_final_evidence_package
write_cvrp_final_evidence_package
```

The service evaluates baseline and candidate workspaces on the same fixed
CVRP case/seed set and converts checked outputs into `QualityCaseRecord` rows.
It then reuses P3-01 to build/write final evidence packages.

### Semantics

- Both sides receive the same case, seed, and time limit.
- Instances are loaded through the CVRP adapter.
- Solver outputs are deserialized through the adapter.
- Solution consistency and feasibility are checked through the adapter.
- Objective is recomputed through the adapter; solver-reported distance is not
  trusted as the final quality value.
- Timeout/crash/oom/error/infeasible/missing-output cases are preserved as
  non-comparable evidence rows.
- Candidate outputs that use more routes than `bks_routes` are benchmark
  incomparable and cannot become fake wins.
- The service does not mutate campaign state and does not make promotion
  decisions.

During main-session review, the generic final-quality writer was tightened:

- CVRP rows with unknown or false benchmark comparability now force
  `comparison="not_comparable"`, even if a caller supplied a better/tie/worse
  comparison.

This prevents a lower-distance but more-vehicles CVRP result from entering final
quality as a win.

## P3-04a CVRP Case Manifest Builder

Added:


```text

CvrpCaseEntry

build_cvrp_case_manifest_from_rows
build_cvrp_case_manifest_from_csv
write_cvrp_case_manifest
```

The builder consumes existing CVRP result CSV rows and creates deterministic
fixed evaluation case manifests. It treats selected case paths as opaque strings
and never opens raw CVRPLIB instance files.

### Semantics

- Supports subset filters such as `("A", "B", "P", "E")` and `("X",)`.
- Supports `require_bks`, `require_bks_routes`, and
  `require_benchmark_feasible`.
- Filters status, feasibility, benchmark comparability, BKS, BKS route counts,
  subset membership, and missing path.
- Selects deterministically with optional total and per-subset caps.
- Selected cases are unique case entries, not expanded by seed.
- Manifest metadata records source label/path, seed list, selected/eligible
  counts, rejected counts, and rejection counts by reason.

## P3-05a Final Evidence Ref Payload Helper

Added:

```text
scion/scion/evidence/final_evidence_refs.py
scion/scion/tests/unit/evidence/test_final_evidence_refs.py
```

Updated:

```text
scion/scion/evidence/__init__.py
scion/docs/planning/v0.4/v0.4-phase3-task-manifest.md
```

Public API:

```text
build_final_evidence_refs
attach_final_evidence_package
```

The helper converts an already-written final evidence package result into a
stable `final_evidence_refs` payload for `EvidenceRecorder`.

### Semantics

- Input is an object with `package` and `artifacts` attributes.
- Artifact `Path` values are converted to strings.
- The payload is nested under a caller-controlled label, default
  `final_quality`.
- Manifest metadata is copied from the in-memory package:
  - schema;
  - package type;
  - problem id;
  - campaign id;
  - baseline label;
  - candidate label;
  - number of cases.
- P3-01 artifact keys are preserved, with missing keys represented as `None`.
- Extra artifact keys are preserved for future evidence package extensions.
- The helper does not read artifact file contents.
- Attaching refs does not change per-step campaign summary schema.
