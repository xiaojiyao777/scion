# 一、总判断

## 1.1 总体结论

**结论：条件通过（Conditional Pass）**。  
我整体上**认可**你们这次 v0.3 审核报告的大方向，尤其下面这些判断，我基本同意：

- W15 必须从“去 warehouse 术语”升级为“建立问题适配边界”
- W2 现在的 async + snapshot 语义不够安全
- W16 当前 18 个 full-system campaigns **答不出**草案声称的全部 RQ
- W5/W8/W9 如果都按现在草案推进，很容易做出新的 memory 技术债
- W7 不能晚于 family persistence 做

如果只给一句非常直接的话：

> **现在这版 v0.3 Draft + Review 可以继续推进，但还不能“按当前文档直接开 Sprint N 实装”。**
> 最可能返工的点只有三个：**W15、W2、W16**。

---

## 1.2 我认为必须马上修正的 4 个设计点

这 4 条我建议你们**不要按 Draft v2 原样实现**：

1. **不要把 objective handling 粗暴改成 `dict[str, float]`**  
   应该 genericize 的是**比较层 / feedback 层**，不是把整个类型契约打散。

2. **不要用 `lower_bound_rule: "|...|"` 这种字符串 DSL**  
   这会把 W15 变成“做 DSL / 做 eval / 做半吊子 parser”，不值当。  
   lower bound 要么常量，要么 adapter hook。

3. **不要机械把 `STALE_WEIGHT_UPDATE` 接上就算 W2 完成**  
   这不是 writer 缺失那么简单，问题本质是：
   - snapshot 要不要原地写
   - stale 是否 stage-aware
   - lineage 如何记录 revision
   - double-promote 如何安全取消

4. **不要让 `toy_tsp/` 只放 README**  
   这会让 W15 的验收变成伪验收。  
   必须有 **minimal working example**。

## P0-05 重做 W2：immutable champion snapshot + revision + stage-aware stale  
**Gate：Sprint N2**

### 目标
让 async weight opt 变成可审计、可取消、不会污染 live snapshot 的机制。

### 当前设计的主要问题
- 原地 chmod + 回写 frozen snapshot 风险高
- `STALE_WEIGHT_UPDATE` 单独 enum 不足以表达恢复语义
- 版本切换没有 revision 语义

### 必做改动
1. structural promote 与 weight-only update 分开版本语义  
   推荐：
   - `version`: 结构版本
   - `revision`: 权重修订号
2. snapshot 采用不可变目录：
   - `champion_v3_r0/`
   - `champion_v3_r1/`
3. champion 使用**原子指针切换**，不原地改写 live snapshot
4. 分支 stale 记录元数据：
   - `stale_reason`
   - `stale_from_stage`
   - `stale_from_version`
   - `stale_from_revision`

### 我建议的更简单建模
**我不建议继续把 `STALE_WEIGHT_UPDATE` 当成核心状态。**  
建议改成：

- 保留 `state=STALE`
- 新增 stale metadata 字段

这样比继续扩 enum 更稳。

### 恢复语义建议
| stale 前阶段 | 最低重做阶段 |
|---|---|
| EXPLORE / EXPLORE_EXPAND | screening |
| READY_VALIDATE / VALIDATING / VALIDATING_EXPAND | validation |
| READY_FROZEN | validation |
| FROZEN_TESTING | 默认不打断，允许当前 frozen 完成；之后再按新 revision stale |

### 验收
- weight update 产生新 snapshot 目录，不改旧目录
- double-promote 时旧后台结果不会污染当前 champion
- stale 恢复按 `stale_from_stage` 生效
- lineage 可追出 `(version, revision)` 变化链路

## 3.3 Pydantic schema 草案

> 说明：下面代码是“接口草案”，不是要求你们 1:1 照抄；  
> 但我建议整体形状尽量保持。

```python
from __future__ import annotations

from dataclasses import dataclass
from enum import Enum
from typing import Any, Literal, Mapping, Protocol, Sequence, runtime_checkable, TypeAlias

from pydantic import BaseModel, ConfigDict, Field, model_validator


MetricValue: TypeAlias = int | float


class StrictModel(BaseModel):
    model_config = ConfigDict(extra="forbid", frozen=True)


class MetricDirection(str, Enum):
    MINIMIZE = "minimize"
    MAXIMIZE = "maximize"


class LowerBoundKind(str, Enum):
    NONE = "none"
    CONSTANT = "constant"
    ADAPTER = "adapter"


class DomainTermSpec(StrictModel):
    name: str
    definition: str


class OperatorCategorySpec(StrictModel):
    name: str
    description: str


class ObjectiveMetricSpec(StrictModel):
    name: str
    direction: MetricDirection = MetricDirection.MINIMIZE
    priority: int = Field(ge=1)
    tolerance_abs: float = 0.0
    tolerance_rel: float = 0.0
    expose_in_feedback: bool = True

    # saturation / lower-bound 相关
    lower_bound_kind: LowerBoundKind = LowerBoundKind.NONE
    lower_bound_constant: MetricValue | None = None

    @model_validator(mode="after")
    def _validate_lower_bound(self) -> "ObjectiveMetricSpec":
        if self.lower_bound_kind == LowerBoundKind.CONSTANT and self.lower_bound_constant is None:
            raise ValueError("lower_bound_constant is required when lower_bound_kind=constant")
        if self.lower_bound_kind != LowerBoundKind.CONSTANT and self.lower_bound_constant is not None:
            raise ValueError("lower_bound_constant must be None unless lower_bound_kind=constant")
        return self


class FamilyLabelSpec(StrictModel):
    label: str
    description: str
    examples: list[str] = Field(default_factory=list)


class FamilyTaxonomySpec(StrictModel):
    version: str
    fallback_label: str = "NEW_FAMILY"
    labels: list[FamilyLabelSpec]

    @model_validator(mode="after")
    def _validate_unique_labels(self) -> "FamilyTaxonomySpec":
        names = [x.label for x in self.labels]
        if len(names) != len(set(names)):
            raise ValueError("family taxonomy labels must be unique")
        if self.fallback_label not in set(names):
            raise ValueError("fallback_label must exist in labels")
        return self


class LLMHintsSpec(StrictModel):
    problem_intro: str
    domain_terms: list[DomainTermSpec] = Field(default_factory=list)
    improvement_axes: list[str] = Field(default_factory=list)
    anti_patterns: list[str] = Field(default_factory=list)


class OperatorInterfaceSpec(StrictModel):
    base_class_import: str
    execute_signature: str = "execute(self, solution, rng) -> Solution"
    categories: list[OperatorCategorySpec]

    @property
    def category_names(self) -> tuple[str, ...]:
        return tuple(x.name for x in self.categories)


class ProblemAdapterRef(StrictModel):
    import_path: str  # e.g. "scion.problems.warehouse_delivery.adapter:WarehouseDeliveryAdapter"
    api_version: Literal["v1"] = "v1"


class ProblemSpecV1(StrictModel):
    spec_version: Literal["problem-v1"] = "problem-v1"

    id: str
    display_name: str
    root_dir: str
    description: str = ""

    # 复用你们现有字段
    search_space: Any
    solver: Any
    parameter_search: Any

    operator_interface: OperatorInterfaceSpec
    objectives: list[ObjectiveMetricSpec]
    llm_hints: LLMHintsSpec
    family_taxonomy: FamilyTaxonomySpec | None = None
    adapter: ProblemAdapterRef

    @model_validator(mode="after")
    def _validate_objectives(self) -> "ProblemSpecV1":
        names = [x.name for x in self.objectives]
        if len(names) != len(set(names)):
            raise ValueError("objective names must be unique")

        priorities = sorted(x.priority for x in self.objectives)
        expected = list(range(1, len(priorities) + 1))
        if priorities != expected:
            raise ValueError(f"objective priorities must be contiguous: got {priorities}, expected {expected}")

        module_name = self.adapter.import_path.split(":")[0]
        expected_prefix = f"scion.problems.{self.id}."
        if not module_name.startswith(expected_prefix):
            raise ValueError(
                f"adapter import_path must live under {expected_prefix}*, got {module_name}"
            )
        return self
```

```python
from dataclasses import dataclass
from typing import Any, Mapping, Protocol, Sequence, runtime_checkable


@dataclass(frozen=True)
class CheckReport:

    objective: Mapping[str, int | float]
    feasible: bool
    normalized_solution: Any | None = None  # adapter-owned opaque object


@runtime_checkable
class ProblemAdapter(Protocol):
    """Problem-specific runtime hooks for Scion core.

    Core 只依赖这个协议，不依赖具体问题的字段名、类名、JSON shape。
    """

    def __init__(self, spec: ProblemSpecV1) -> None: ...

    # ---------- Prompt / context ----------
    def render_problem_summary(self) -> str:
        """Return a rich problem summary for Round-1/2 prompt construction."""
        ...

    def render_operator_interface(self) -> str:
        """Return operator base class / interface / key invariants as prompt text."""
        ...

    # ---------- Instance / output ----------
    def load_instance(self, instance_path: str) -> Any:
        """Load one benchmark instance into a problem-specific object."""
        ...

    def deserialize_solver_output(
        self,
        raw_output: Mapping[str, Any],
        instance: Any,
    ) -> SolverArtifact:
        """Convert raw solver JSON output into a normalized artifact."""
        ...

    # ---------- Verification ----------
    def check_solution_consistency(
        self,
        artifact: SolverArtifact,
        instance: Any,
    ) -> CheckReport:
        """Problem-specific internal consistency check.
        Example:
          - assignment vs vehicle.order_ids consistency
          - tour permutation validity
        """
        ...

    def check_feasibility(
        self,
        artifact: SolverArtifact,
        instance: Any,
    ) -> CheckReport:
        """Hard feasibility check against frozen oracle / constraints."""
        ...

    def recompute_objective(
        self,
        artifact: SolverArtifact,
        instance: Any,
    ) -> Mapping[str, int | float]:
        """Recompute objective from artifact + instance for verification."""
        ...

    # ---------- Saturation / lower bound ----------
    def estimate_lower_bound(
        self,
        metric_name: str,
        instance_paths: Sequence[str],
    ) -> LowerBoundEstimate | None:
        """Optional stronger lower-bound estimate for saturation analysis.
        Return None if not implemented; core falls back to constant lower bound or no bound.
        """
        ...
```

## 3.5 generic objective comparator 草案

这个部分我建议放在 `scion/problem/objectives.py`，由 core 统一调用。  
这样就不需要把所有 objective 到处改成裸 dict。

```python
from __future__ import annotations

from dataclasses import dataclass
from typing import Literal, Mapping, Sequence


@dataclass(frozen=True)
class MetricComparison:
    name: str
    candidate_value: int | float
    champion_value: int | float
    signed_delta: float   # >0 means candidate better under the metric's direction
    relation: Literal["candidate", "champion", "tie"]
    decisive: bool = False


@dataclass(frozen=True)
class ObjectiveComparison:
    outcome: Literal["win", "loss", "tie"]
    decisive_metric: str | None
    scalar_delta: float
    metrics: tuple[MetricComparison, ...]


def compare_lexicographic(
    metric_specs: Sequence[ObjectiveMetricSpec],
    candidate: Mapping[str, int | float],
    champion: Mapping[str, int | float],
) -> ObjectiveComparison:
    ordered = sorted(metric_specs, key=lambda x: x.priority)

    rows: list[MetricComparison] = []
    decisive_metric: str | None = None
    outcome: Literal["win", "loss", "tie"] = "tie"
    scalar_delta = 0.0

    for spec in ordered:
        c = float(candidate[spec.name])
        h = float(champion[spec.name])

        tol = max(spec.tolerance_abs, abs(h) * spec.tolerance_rel)

        # signed_delta > 0 means candidate is better
        if spec.direction == MetricDirection.MINIMIZE:
            signed_delta = h - c
        else:
            signed_delta = c - h

        if abs(signed_delta) <= tol:
            relation: Literal["candidate", "champion", "tie"] = "tie"
        elif signed_delta > 0:
            relation = "candidate"
        else:
            relation = "champion"

        is_decisive = decisive_metric is None and relation != "tie"
        if is_decisive:
            decisive_metric = spec.name
            scalar_delta = signed_delta
            outcome = "win" if relation == "candidate" else "loss"

        rows.append(
            MetricComparison(
                name=spec.name,
                candidate_value=c,
                champion_value=h,
                signed_delta=signed_delta,
                relation=relation,
                decisive=is_decisive,
            )
        )

    return ObjectiveComparison(
        outcome=outcome,
        decisive_metric=decisive_metric,
        scalar_delta=scalar_delta,
        metrics=tuple(rows),
    )
```

## 3.6 安全加载器草案

```python
from __future__ import annotations

import importlib
from typing import cast


class ProblemAdapterLoadError(RuntimeError):
    pass


def load_problem_adapter(spec: ProblemSpecV1) -> ProblemAdapter:
    try:
        module_name, class_name = spec.adapter.import_path.split(":")
    except ValueError as exc:
        raise ProblemAdapterLoadError(
            f"invalid adapter import_path: {spec.adapter.import_path}"
        ) from exc

    expected_prefix = f"scion.problems.{spec.id}."
    if not module_name.startswith(expected_prefix):
        raise ProblemAdapterLoadError(
            f"adapter must live under {expected_prefix}*, got {module_name}"
        )

    module = importlib.import_module(module_name)
    cls = getattr(module, class_name, None)
    if cls is None:
        raise ProblemAdapterLoadError(
            f"class {class_name} not found in module {module_name}"
        )

    adapter = cls(spec)
    if not isinstance(adapter, ProblemAdapter):
        raise ProblemAdapterLoadError(
            f"{spec.adapter.import_path} does not implement ProblemAdapter"
        )
    return cast(ProblemAdapter, adapter)
```

### 为什么这个设计不违背 Contract “禁止 shell / subprocess”的精神
因为这里：
- 只导入**problem-root 下受控模块**
- import path 是 frozen spec 的一部分
- 不是 LLM patch 任意指定的模块
- 可以被 spec hash / lineage 记录
