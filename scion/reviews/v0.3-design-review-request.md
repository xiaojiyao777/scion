## 1. 项目速览

### 1.1 Scion 是什么

**一句话**：面向组合优化算法的自动研究框架——让"提假设 → 改算子 → 验证 → 统计判断 → 晋升/淘汰"的过程可信、可控、可追溯。

**不是什么**：
- 不是通用 agent 框架
- 不是 FunSearch-style 的候选程序生成器
- 不是 LLM 进化算法（LLM 作为变异算子）
- 不自动定义问题，不做从零设计

**核心控制模型**：
- **三层控制**：Creative(LLM, tainted) → Contract Gate → Verification Gate → Decision Layer
- **Decision Input Guard**：DecisionFeatures 只含数值 + 枚举，零自由文本
- **两轮 Proposal**：Round 1 Hypothesis → Round 2 Code
- **三级实验协议**：Screening → Validation → Frozen Holdout
- **Champion 池级别**：不是单算子，而是算子池配置

### 1.2 核心差异点

| # | 维度 | FunSearch / EoH / ReEvo | Scion |
|---|---|---|---|
| 1 | 分支治理 | 无（种群式） | explore→validate→promote 状态机 |
| 2 | 验证分离 | 无（评估即验证） | Contract + Verification + Protocol 三关独立 |
| 3 | 信息隔离 | 无 | 三层控制 + Decision Input Guard |
| 4 | 统计协议 | 点估计 keep/discard | 三级 split + seed ledger + 暴露控制 + bootstrap CI |
| 5 | 可追溯性 | 弱或无 | 完整 lineage，hypothesis → code → eval → decision 全链路 |

### 1.3 当前 Scope

- **目标问题**：仓配协同优化（warehouse_delivery），基于 VNS + Solution Pool 的启发式求解器
- **代码规模**：Scion 核心 ~11,400 行 Python / 289 unit tests / v0.2 归档于 commit `cf50429`
- **运行数据**：v0.2 完成 6 轮正式实验（F1-F6），合成 + 生产数据均有覆盖

### 3.3 v0.2 发现的核心问题（直接驱动 v0.3 设计）

#### 问题 1：生产数据 weight opt 完全失效

- 合成数据 splits delta 量级 O(100K) → 权重优化空间清晰 → 3/3 improved

- 根因：`compute_delta()` 被**实验协议**和 **weight optimization** 共用同一 scoring

#### 问题 2：未接通的功能骨架（v0.2-final-state.md 发现 17 处 plumbing gap，关键 5 处）

- **D01**：`HypothesisFamilyClassifier` 只在 tests 里实例化，主循环不调用
- **D02**：`HypothesisRecord.family_id` 字段存在但 SQLite 表无列
- **D03**：`CampaignSearchMemory` 完全 in-process，不持久化
- **D04**：`BranchState.STALE_WEIGHT_UPDATE` 定义存在但无 writer（async weight opt 骨架没真正接通）
- **D08**：`experiment_events` 表 prompt/completion_tokens 列建了但不写

#### 问题 3：Scion 核心与 warehouse_delivery 深度耦合

- `proposal/context_manager.py` prompt 里 hard-code 领域术语
- `proposal/saturation.py` hard-code "splits 有绝对下界"
- `verification/feasibility.py` 硬依赖 `surrogate.oracle`
- `protocol/evaluation.py:compute_delta` hard-code `(splits, cost, runtime)` 元组

#### 问题 4：搜索停滞后浪费预算

- F4-A R57 最后一次 promote → 后 41 轮零晋升，跑满 200r
- F4-B R4 最后一次 promote → 后 96 轮零晋升

## 7. 关键术语表（自检用）

| 术语 | 含义 |
|---|---|
| Champion | 当前最佳的算子池配置（池级别，不是单算子） |

| DecisionFeatures | 白名单输入（只含数值 + 枚举），Decision Layer 唯一可读数据 |
| STALE | 分支状态：champion 推进后活跃分支被标记，需要 reconcile |

| plateau | v0.3 新引入的"搜索停滞"信号，用于早停 |
