## 0. 定位

以 autoresearch 方法论为内核的专用 agent 框架，面向组合优化算法自动设计。

- 接受已定义好的组合优化问题（baseline + benchmark + 评价标准）
- 按科学方法论自主执行算法探索与优化
- 通过统计验证输出可靠的、优于 baseline 的新算法

**不是通用 agent 框架。** 不是 LangChain/MiroFlow 的竞品。

---

## 1. 核心原则

1. **方法论是内核，agent 是执行器** — 框架价值在搜索治理方法论
2. **确定性逻辑用代码，创造性推理用 LLM** — 分支控制器/统计门槛/验证管线全确定性
3. **三层控制模型** — LLM 创造 → Contract 约束 → Decision 决策（见 §4）
4. **人在回路但不在循环里** — 人定义问题+审核结果，agent 独立执行探索循环

## 3. v2 新增模块设计

### 3.1 Experiment Protocol（实验协议层）

**v1 问题**：Promotion Gate 只是决策规则，不是实验协议。缺少 seed 策略、采样策略、回归检测。

```python
class ExperimentProtocol:
    """
    定义"如何做一次可信实验"的完整协议。
    与 PromotionGate 的关系：Protocol 产出数据，Gate 做决策。
    """

    def __init__(self, config):
        self.n_cases = config.n_cases          # ≥6
        self.seeds = config.seeds              # 固定 seed 列表
        self.time_limit = config.time_limit_sec
        self.metric = config.metric
        self.higher_is_better = config.get("higher_is_better", False)
        self.min_practical_delta = config.min_practical_delta

    def sample_cases(self, instance_pool, exclude_used=None):
        """从 instance pool 中采样 n_cases 个，排除已用过的（防过拟合）"""
        available = [i for i in instance_pool if i not in (exclude_used or set())]
        return random.sample(available, min(self.n_cases, len(available)))

    def run_paired_evaluation(self, candidate_code, champion_code, case_set):
        """
        配对评估：同 case 同 seed，candidate vs champion。
        返回结构化结果（不是标量）。
        """
        results = []
        for case in case_set:
            for seed in self.seeds:
                cand_score = self.tools.benchmark_run(
                    candidate_code, case, seed, self.time_limit
                )
                champ_score = self.tools.benchmark_run(
                    champion_code, case, seed, self.time_limit
                )
                results.append(PairedResult(
                    case_id=case.id, seed=seed,
                    candidate_score=cand_score,
                    champion_score=champ_score
                ))
        return PairedEvaluation(results)

    def evaluate_with_gate(self, paired_eval):
        """Protocol 产出数据 → Gate 做决策"""
        # 聚合为 per-case best（跨 seed）
        case_deltas = paired_eval.aggregate_per_case(self.higher_is_better)
        return promotion_gate(
            case_deltas.candidate_scores,
            case_deltas.champion_scores,
            self.min_practical_delta
        )

    def check_regression(self, candidate_code, regression_cases):
        """回归检测：candidate 在历史 champion case 上不能显著退化"""
        # v0.1 简化版：只检查 feasibility + 不超过 champion 10%
        pass
```

### 3.2 Contract Layer（合约层 — LLM 输出约束）

**v1 问题**：LLM 可以自由改代码，没有硬约束搜索半径。

```python
class ContractLayer:
    """
    夹在 LLM Creative Layer 和 Deterministic Decision Layer 之间。
    所有 LLM 输出必须经过 Contract 验证才能进入系统状态。
    """

    def __init__(self, problem_spec):
        self.editable_files = problem_spec.search_space.editable_files   # 白名单
        self.frozen_files = problem_spec.search_space.frozen_files       # 黑名单
        self.interface_spec = problem_spec.search_space.interface
        self.max_patch_lines = problem_spec.get("max_patch_lines", 200)

    def validate_code_patch(self, original_code, new_code) -> ContractResult:
        """代码补丁合约检查"""
        checks = []

        # 1. 文件白名单
        checks.append(self._check_file_whitelist(new_code))

        # 2. Patch 尺寸限制
        diff_lines = self._compute_diff_lines(original_code, new_code)
        checks.append(self._check_patch_size(diff_lines))

        # 3. 接口合规（AST 检查）
        checks.append(self._check_interface_compliance(new_code))

        # 4. 禁止修改 benchmark harness / 评估代码
        checks.append(self._check_frozen_files_untouched(new_code))

        # 5. 语法正确性
        checks.append(self._check_syntax(new_code))

        return ContractResult(checks)

    def validate_hypothesis_proposal(self, proposal) -> ContractResult:
        """假设提案的 schema 验证"""
        required_fields = ["hypothesis_text", "target_weakness",
                          "expected_improvement", "change_locus"]
        return self._check_schema(proposal, required_fields)

    def validate_analysis_proposal(self, analysis) -> ContractResult:
        """结果分析的 schema 验证 — 防止 LLM 间接控制决策"""
        allowed_fields = ["suspected_failure_mode", "improvement_axes",
                         "confidence", "evidence_summary"]
        return self._check_schema_strict(analysis, allowed_fields)
```

### 3.3 Runtime Isolation（执行隔离）

**v1 问题**：分支间可能文件污染、cache 误复用。

```python
class RuntimeIsolation:
    """
    每个 branch 独立工作目录，champion 快照不可变。
    v0.1 简化实现：用目录隔离代替 Docker。
    """

    def __init__(self, workspace_root):
        self.workspace_root = workspace_root
        self.champion_dir = workspace_root / "champion"  # 不可变快照

    def create_branch_workspace(self, branch_id, champion_snapshot):
        """从 champion 快照创建分支工作目录"""
        branch_dir = self.workspace_root / f"branches/{branch_id}"
        shutil.copytree(champion_snapshot, branch_dir)
        return branch_dir

    def freeze_champion(self, code, version):
        """晋升后冻结新 champion"""
        snapshot_dir = self.workspace_root / f"champion_v{version}"
        # 写入代码 + 锁定
        self._write_and_lock(snapshot_dir, code)
        # 更新 champion 符号链接
        self.champion_dir.unlink()
        self.champion_dir.symlink_to(snapshot_dir)

    def cleanup_branch(self, branch_id):
        """abandon 后清理分支目录"""
        branch_dir = self.workspace_root / f"branches/{branch_id}"
        shutil.rmtree(branch_dir, ignore_errors=True)

    def run_in_isolation(self, branch_id, command, timeout):
        """在分支目录中执行命令，超时控制 + 输出截断"""
        branch_dir = self.workspace_root / f"branches/{branch_id}"
        return subprocess.run(
            command, cwd=branch_dir,
            timeout=timeout, capture_output=True,
            text=True
        )
```

### 3.5 Failure Taxonomy（失败分类体系）

**v1 问题**："3次重试"不区分失败类型。

```python
class FailureType(Enum):
    # 可修复 — 反馈 LLM 重试
    SYNTAX_ERROR = "syntax_error"
    INTERFACE_VIOLATION = "interface_violation"
    CONTRACT_VIOLATION = "contract_violation"

    # 不可修复 — 直接丢弃本次
    RUNTIME_TIMEOUT = "runtime_timeout"
    RUNTIME_OOM = "runtime_oom"
    FEASIBILITY_VIOLATION = "feasibility_violation"

    # 基础设施问题 — 不惩罚分支
    BENCHMARK_INFRA_FAILURE = "benchmark_infra_failure"
    LLM_API_FAILURE = "llm_api_failure"

    # 结构性问题 — 记录到 hypothesis failure
    PERFORMANCE_REGRESSION = "performance_regression"
    HYPOTHESIS_INEFFECTIVE = "hypothesis_ineffective"

class FailureHandler:
    RETRY_ELIGIBLE = {
        FailureType.SYNTAX_ERROR,
        FailureType.INTERFACE_VIOLATION,
        FailureType.CONTRACT_VIOLATION,
    }
    DISCARD_ELIGIBLE = {
        FailureType.RUNTIME_TIMEOUT,
        FailureType.RUNTIME_OOM,
        FailureType.FEASIBILITY_VIOLATION,
    }
    INFRA_FAILURES = {
        FailureType.BENCHMARK_INFRA_FAILURE,
        FailureType.LLM_API_FAILURE,
    }

    def handle(self, failure_type, branch, context):
        if failure_type in self.RETRY_ELIGIBLE:
            return Action.RETRY_WITH_FEEDBACK
        elif failure_type in self.DISCARD_ELIGIBLE:
            return Action.DISCARD_THIS_ATTEMPT
        elif failure_type in self.INFRA_FAILURES:
            return Action.PAUSE_AND_RETRY_LATER  # 不消耗分支预算
        else:
            return Action.RECORD_AND_CONTINUE
```

### 3.6 终止条件（v1 只有 `while budget`）

```python
def should_continue(self):
    """多重终止条件"""
    # 硬预算
    if self.experiment_log.total_experiments >= self.spec.budget.max_experiments:
        return False
    if self.experiment_log.wall_clock_hours >= self.spec.budget.max_wall_clock_hours:
        return False

    # 停滞检测
    if self.branch_controller.consecutive_global_failures >= 5:
        return False  # 连续 5 个分支全部 abandoned，搜索空间可能穷尽

    # 无活跃分支且无法创建新分支
    if (self.branch_controller.active_count == 0
        and not self._can_create_new_branch()):
        return False

    return True
```

## 4. 三层控制模型（v2 核心升级）

**v1 只定义了"LLM 的 4 个介入点"。v2 升级为三层约束模型。**

```
Layer A: LLM Creative Layer
  ├── hypothesis proposal     → 必须经过 schema 验证
  ├── code patch proposal     → 必须经过 contract 检查
  ├── failure analysis        → 必须经过 allowed_fields 过滤
  └── branch direction        → 必须经过 novelty 检查
          │
          ▼
Layer B: Contract Layer（确定性约束）
  ├── JSON schema validation
  ├── file whitelist enforcement
  ├── AST / interface compliance
  ├── patch size limit
  ├── forbidden-file check
  └── analysis field filter
          │
          ▼
Layer C: Decision Layer（确定性决策）
  ├── branch state transitions
  ├── scheduling
  ├── promotion / abandonment
  ├── termination
  └── artifact persistence
```

**边界清晰度**：LLM 产出的不是"决策"而是"提案"，提案必须通过 Contract 才能进入 Decision。

## 5. 调度策略形式化（v2 新增）

```python
class Scheduler:
    """显式调度策略，不再是隐式 heuristic"""

    def select(self) -> Optional[Branch]:
        # Priority 1: 解决 validation debt
        validate_branches = self._get_validate_branches()
        if validate_branches:
            return self._oldest_first(validate_branches)

        # Priority 2: 推进最有希望的 explore
        explore_branches = self._get_explore_branches()
        if explore_branches:
            return self._rank_by_signal(explore_branches)
            # signal = (best_gate_label, median_delta, -consecutive_failures)

        # Priority 3: 创建新分支（需要 LLM 提出新方向）
        if self._can_create_new():
            return None  # 信号给主循环：需要创建新分支

        return None  # 终止信号

    def _can_create_new(self):
        return (
            self.active_count < self.max_active
            and not self._has_pending_validate()
            and self.remaining_budget > self.min_branch_budget
        )
```

## 6. 结构化假设记忆（v2 升级 Context Manager）

```python
@dataclass
class HypothesisRecord:
    """结构化假设对象，替代自然语言日志"""
    hypothesis_id: str
    hypothesis_text: str
    change_locus: str          # "destroy operator", "acceptance criterion", etc.
    expected_effect: str       # "improve dense-urban by reducing fragmentation"
    observed_effect: str       # "dense-urban +1.2%, but sparse -0.5%"
    failure_mode: str          # "partial_improvement", "regression", "no_effect"
    confidence: float          # 0-1
    blacklist_scope: str       # "local" (this direction) or "global" (don't try again)

class ContextManager:
    def build(self, branch, experiment_history, champion_info, problem_spec):
        context = []
        # Layer 1: 问题定义（始终完整）
        context.append(self._format_problem_spec(problem_spec))
        # Layer 2: Champion 状态
        context.append(self._format_champion(champion_info))
        # Layer 3: 当前分支历史（近 5 轮详细，更早压缩）
        context.append(self._format_branch_history(experiment_history))
        # Layer 4: 结构化失败假设（按 blacklist_scope 过滤）
        context.append(self._format_failed_hypotheses_structured())
        # Layer 5: 兄弟分支摘要
        context.append(self._format_sibling_branches())
        return AgentContext(context)
```

## 9. 差异化定位（v2 精化）

> **FunSearch/EoH 优化的是"候选程序生成与筛选"；**
> **本框架优化的是"研究过程本身的结构化治理"。**

5 个结构性差异点：

| # | 差异点 | FunSearch/EoH/ReEvo | 本框架 |
|---|--------|-------------------|--------|
| 1 | 假设对象 | 隐式（代码即假设） | 显式 HypothesisRecord，可审计可复用 |
| 2 | 分支治理 | 无（单线或种群） | explore→validate→promote 状态机 |
| 3 | 验证分离 | 无（评估即验证） | frozen code + fresh cases 二阶段 |
| 4 | 统计门槛 | 点估计选择 | win_rate + median_delta + retry protocol |
| 5 | 失败记忆 | 无或弱 | 结构化 failure taxonomy + blacklist |

vs Karpathy autoresearch：
> 从"单线程代码代理"提升为"实验分支治理系统"。

6. mock LLM 后确定性部分独立运转
